================================================================================
  D&D 2024 CHARACTER SHEET — FUTURE IMPLEMENTATION PLAN
  Rest System, Weapon Mastery, Class Features, and All Character Options
================================================================================

This document serves as a comprehensive blueprint for extending the rest system,
weapon mastery, and class/species/background features to every class, subclass,
species, background, and item in the app. It is written to be applied by a
future developer (or AI instance) with full context of the codebase architecture.

--------------------------------------------------------------------------------
ARCHITECTURE OVERVIEW (READ FIRST)
--------------------------------------------------------------------------------

Key files and responsibilities:
  - js/data.js        : Data loading (5etools API + UA2024_DATA), helper
                        functions (getClassInfo, getSpeciesInfo, etc.)
  - js/ua-data.js     : Bundled UA/2024 data for classes/species not yet in
                        the 5etools fetch (add new content here)
  - js/wizard.js      : 8-step character creation wizard
  - js/sheet.js       : Main character sheet rendering and logic
  - js/combat.js      : Combat tab, hit dice, and REST MODAL SYSTEM
  - js/levelup.js     : Level-up modal (ASIs, subclass, HP, new features)
  - js/store.js       : CharStore — localStorage per-character storage

Character data is stored in localStorage as JSON keyed by character ID.
Key fields relevant to this plan:
  - charClass, charSubclass, charSpecies, charBackground, charLevel
  - resources: [{name, max, used, refresh}]   — limited use features
  - weaponMastery: [string, ...]              — chosen mastery weapons
  - hitDice: [{die, total, used}]             — hit dice tracking
  - charSpells: [{name, level, prepared}]     — known/prepared spells

The REST MODAL SYSTEM (js/combat.js) is the central hub for all rest logic.
It has three entry points:
  _openRestModal('short')  → Short Rest modal
  _openRestModal('long')   → Long Rest modal
  _openRestModal('dawn')   → Dawn modal

Each modal type calls a body-builder function and a do-execute function.

SECTION 2: REST SYSTEM — COMPLETE CLASS-BY-CLASS GUIDE
================================================================================

The rest modal system is in js/combat.js. The three modal types and what each
class should offer within them:

─────────────────────────────────────────────────
SHORT REST — _buildShortRestBody / _doShortRest
─────────────────────────────────────────────────
Features that refresh on Short Rest (refresh: 'sr'):

  BARBARIAN   : Nothing special — Rage does NOT refresh on short rest.
  BARD        : Bardic Inspiration (starting at level 5 onward it's LR;
                before level 5 it refreshes on short rest — Bard 2014 only).
  FIGHTER     : Second Wind (1 use, refresh SR), Action Surge (1–2 uses SR),
                Indomitable (LR). Fighter 2024: Second Wind restores HP.
  MONK        : Discipline Points (all expended points recovered on SR),
                Stunning Strike (uses Discipline Points).
  RANGER      : Nothing specifically SR in 2024.
  WARLOCK     : Spell Slots (ALL warlock spell slots recover on SR) — this
                is THE most important SR feature. In _doShortRest, detect if
                charClass === 'Warlock' and restore all warlock spell slots.
  WIZARD      : Arcane Recovery (short rest feature — recover some spell slots
                once per day). This is once-per-LR, usable during SR.

  IMPLEMENTATION NOTES:
  - Warlock slot recovery: in _doShortRest(), check charClass. If Warlock,
    restore all spell slots (same logic as LR slot restore in _doLongRest).
  - Wizard Arcane Recovery: store as a resource with refresh='lr'. It appears
    in the SR modal because it's used during SR. To handle: add a flag
    'usableOnShortRest: true' to such resources, and show them in the SR modal
    under "Once-per-day features usable during Short Rest."
  - Generally, auto-populate class resources on character creation/level-up
    (see Section 4 of this document).

─────────────────────────────────────────────────
LONG REST — _buildLongRestBody / _doLongRest
─────────────────────────────────────────────────
Features that refresh on Long Rest (refresh: 'lr'):

  ALL CLASSES   : HP to max, half hit dice restored (min 1 per type),
                  all spell slots restored, death saves reset,
                  exhaustion -1. Already implemented.

  BARBARIAN     : Rage uses (all), Weapon Mastery swap (1 weapon).
  BARD          : Bardic Inspiration (lv 5+), Magical Secrets uses.
  CLERIC        : Channel Divinity (1–3 uses depending on level),
                  Divine Intervention (1 use).
  DRUID         : Wild Shape (uses), Wild Strikes, Beast Spells.
  FIGHTER       : Indomitable (1–3 uses). Second Wind / Action Surge already
                  refresh on SR so they restore here too (LR includes SR).
  MONK          : Discipline Points already LR (all restored), Perfect Self,
                  Empty Body.
  PALADIN       : Lay on Hands (HP pool restored fully), Channel Divinity,
                  Divine Smite slots (all spell slots restore).
  RANGER        : Foe Slayer, some subclass features.
  ROGUE         : Reliable Talent (passive, no tracking), Uncanny Dodge
                  (passive), Evasion (passive). Stroke of Luck (1 use LR).
  SORCERER      : Sorcery Points (all), Metamagic, Innate Sorcery.
  WARLOCK       : Mystic Arcanum (1 use per level 6/7/8/9 — LR, NOT SR).
  WIZARD        : Signature Spells (once per SR but reset on LR).

  LONG REST OPTIONAL CHOICES:
  Some classes make choices during a Long Rest:
  - Barbarian: Swap one Weapon Mastery choice (ALREADY IMPLEMENTED).
  - Druid: Choose Wild Shape form (if rule is form-change-on-LR).
  - Wizard: Change prepared spells — in the LR modal, show a link/note
    directing the player to the Spells tab to update prepared spells.
    Flag prepared spells as "prepared spells may be changed after LR."
  - Cleric/Paladin/Druid: Prepared spells change — same note as Wizard.
  - Any class with "practice weapon drills" (Weapon Mastery): show the
    mastery swap UI.

─────────────────────────────────────────────────
DAWN — _buildDawnBody / _doDawn
─────────────────────────────────────────────────
Features that refresh at dawn (refresh: 'dawn'):

  CLERIC        : Divine Intervention recharges at dawn in some variants.
  DRUID         : Some Wild Shape forms list "recharge at dawn."
  ITEMS         : Many magic items recharge at dawn:
                    - Ring of Three Wishes (some interpretations)
                    - Staff of the Magi (recharges 2d6+4 charges at dawn)
                    - Wand of Fireballs (recharges 1d6+1 charges at dawn)
                    - Deck of Many Things (optional)
                    - Most "X charges, regains Y at dawn" items
  PATH OF THE BERSERKER (Barbarian subclass level 10: Retaliation):
                  Retaliation is a reaction, not a tracked resource.
                  NOT dawn-refresh.
  PATH OF WILD MAGIC (Barbarian subclass):
                  Wild Surge (no per-rest tracking — random on rage).

  IMPLEMENTATION:
  Dawn resources are already supported (refresh: 'dawn'). The key task is
  auto-populating magic item charges as dawn resources when items are added
  to inventory. See Section 5 of this document.

SECTION 3: BARBARIAN — COMPLETE FEATURE CHECKLIST
================================================================================

When a Barbarian character is created or levels up, the following resources
and features should be auto-populated in the character's `resources` array.

AUTO-ADD ON CHARACTER CREATION (level 1):
  - Rage:
      name: "Rage", max: 2, refresh: 'lr'
      (Note: max scales with level — see table below)
  - Weapon Mastery: HANDLED via weaponMastery[] array, not resources.

RAGE USES TABLE (XPHB 2024):
  Level 1–2:   2 uses,  +2 damage
  Level 3–5:   3 uses,  +2 damage
  Level 6–8:   4 uses,  +2 damage
  Level 9–11:  4 uses,  +3 damage
  Level 12–16: 5 uses,  +3 damage
  Level 17–19: 6 uses,  +4 damage
  Level 20:    Unlimited (set max to 99 or add special flag)

WEAPON MASTERY TABLE:
  Level 1–3:  2 weapons
  Level 4–9:  3 weapons
  Level 10+:  4 weapons

FEATURES BY LEVEL (no resource tracking needed — informational only):
  Level 1:  Rage (tracked), Unarmored Defense, Weapon Mastery (tracked)
  Level 2:  Danger Sense (passive), Reckless Attack (passive)
  Level 3:  Barbarian Subclass, Primal Knowledge (+1 skill)
  Level 4:  Ability Score Improvement, Weapon Mastery +1 slot
  Level 5:  Extra Attack (passive), Fast Movement (passive)
  Level 6:  Subclass feature, Rage Uses +1 (now 4)
  Level 7:  Feral Instinct (passive), Instinctive Pounce (passive)
  Level 8:  Ability Score Improvement
  Level 9:  Brutal Strike (uses Reckless Attack — no separate tracking)
  Level 10: Subclass feature, Weapon Mastery +1 slot (now 4)
  Level 11: Relentless Rage (passive — no resource use)
  Level 12: Ability Score Improvement, Rage Uses +1 (now 5)
  Level 13: Improved Brutal Strike (Hamstring Blow — no resource)
  Level 14: Subclass feature
  Level 15: Persistent Rage (passive — rage won't end on low roll)
  Level 16: Ability Score Improvement
  Level 17: Improved Brutal Strike (Staggering/Sundering), Rage Uses +1 (6)
  Level 18: Indomitable Might (passive — STR check minimum = STR score)
  Level 19: Epic Boon
  Level 20: Primal Champion (+4 STR/+4 CON), Unlimited Rage

PATH OF THE BERSERKER (Subclass):
  Level 3:  Frenzy
              - During Rage, bonus action attack each turn (no extra resource;
                it's an extension of Rage)
              - After Rage: 1 level of Exhaustion (auto-track: when Rage ends
                after using Frenzy, increment exhaustion by 1)
              AUTO-ADD resource: "Frenzy (active this Rage): max 1, refresh SR
              (cleared when rage ends)" — or handle via UI checkbox on rage
  Level 6:  Mindless Rage
              - Immunity to Charmed/Frightened while Raging — passive, no track
  Level 10: Retaliation
              - Reaction attack when creature damages you — passive, no track
  Level 14: Intimidating Presence
              AUTO-ADD resource: "Intimidating Presence": max 1, refresh LR
              (Can frighten a creature; once used, must LR to use again
               — unless the target is immune)

IMPLEMENTATION STEPS FOR BARBARIAN AUTO-POPULATION:

1. In wizard.js _finalize(), after class is confirmed as 'Barbarian', call
   a helper function addClassStartingResources(draft) that reads the class
   feature data and pushes default resources into draft.resources.

2. In levelup.js, when confirming a level-up, call
   updateClassResources(charClass, charSubclass, newLevel) which:
     a. Updates existing class resources (e.g., Rage max from 2 → 3 at lv3)
     b. Adds new resources for newly gained features
     c. Does NOT duplicate resources already present (check by name)

3. Store rage damage bonus separately for display:
     CharStore.sv('rageDamageBonus', 2)  // or 3, 4 based on level
   Show it in the combat stats or as a note in the Rage resource.

  In wizard.js _finalize(), after applying species data, call:
    addSpeciesStartingResources(draft)
  This reads the species entries text, detects limited-use patterns
  (phrases like "once per Long Rest", "number of times equal to your
  proficiency bonus"), and adds the corresponding resource entry.

  Pattern detection heuristics (search feature text):
    "once" + ("long rest" OR "LR") → max: 1, refresh: 'lr'
    "once" + ("short rest" OR "SR") → max: 1, refresh: 'sr'
    "at dawn" → refresh: 'dawn'
    "proficiency bonus" + "per" → max: profBonus (dynamic, update on level-up)
    "number of times equal to your [Ability] modifier" → max: abilityMod

SECTION 7: MAGIC ITEMS — AUTO-POPULATION GUIDE
================================================================================

When a magic item with charges is added to inventory, auto-create a dawn
resource entry for it.

COMMON PATTERNS in item text:
  "X charges" + "regains" + Y + "charges" + "at dawn"
  → {name: "[Item Name] Charges", max: X, used: 0, refresh: 'dawn'}

  "once per day" or "once per long rest"
  → {name: "[Item Name]", max: 1, refresh: 'lr'}

  "once per short rest"
  → {name: "[Item Name]", max: 1, refresh: 'sr'}

SPECIFIC ITEMS TO HANDLE:
  Staff of the Magi:     max: 50 charges, dawn recovery (2d6+4)
                         → store as: {name: "Staff of the Magi", max: 50,
                                      refresh: 'dawn', rechargeRoll: '2d6+4'}
  Wand of Fireballs:     max: 7 charges, dawn: 1d6+1
  Rod of Lordly Might:   max: 6 charges, dawn: 1d6
  Cloak of Invisibility: max: 2 hours use, dawn refresh
  Ring of Regeneration:  passive (no tracking)
  Amulet of Health:      passive
  Bag of Holding:        no charges

IMPLEMENTATION STEPS:
1. When adding an item to inventory (sheet.js addInventoryItem or inventory modal),
   check item._isMagic and item.entries text for the patterns above.
2. If a charge pattern is detected, prompt: "Add [X] charges as a tracked
   resource?" with a Yes/No dialog.
3. If Yes, push to resources array with appropriate refresh type.
4. In the dawn modal, display all resources with refresh: 'dawn' and show
   the rechargeRoll if present (show a dice button to roll recovery).

ATTUNEMENT:
  - Track attuned items as CharStore.sv('attunedItems', ['item1', 'item2', 'item3'])
  - Maximum 3 attuned items (unless a feature allows more).
  - In the Long Rest modal, show current attuned items with checkboxes to
    change attunement (add/remove up to 3 slots).
  - In inventory, add an "Attune" button next to magic items that require it.

SECTION 9: LEVEL-UP INTEGRATION
================================================================================

In levelup.js, when a level-up is confirmed, the system should:

1. DETECT RESOURCE CHANGES
   Call getClassStartingResources(className, subclassName, newLevel) and compare
   against current resources. For each new/changed resource:
     a. New feature: push to resources array
     b. Updated max (e.g. Rage 2→3): find by name and update max
     c. Changed refresh type: update refresh field

2. DETECT WEAPON MASTERY THRESHOLD CROSSED
   If newLevel is 4, 5, or 10 (class-dependent), show a mastery-weapon picker
   step in the level-up modal so the player chooses the new weapon slot.

3. DETECT SUBCLASS ACQUISITION (level 3 for most classes)
   When subclass is chosen, call getSubclassStartingResources(subclassName, 3)
   and add subclass resources (e.g. Berserker: Intimidating Presence at lv14).

4. RAGE DAMAGE BADGE
   Store CharStore.sv('rageDamageBonus', N) and display it on the sheet.
   Update it on every Barbarian level-up using the table in Section 3.

================================================================================
NOTES ON DATA SOURCES
================================================================================

- 5etools JSON is the primary source for class features, spell data, items.
  Base URL: https://raw.githubusercontent.com/5etools-mirror-3/5etools-src/main/data/
  Class files: data/class/class-{barbarian|bard|cleric...}.json
  Each class JSON has:
    class[0].classFeature  → array of features with level, name, entries
    class[0].classTableGroups → the progression table (includes mastery column)

- For weapon mastery counts per level, parse classTableGroups looking for
  the column labelled "Weapon Mastery" and read the value at each level.

- For checking if a weapon has a mastery property: DndData.allItems
  item.mastery = ['Topple|XPHB']  → strip source tag for display.
